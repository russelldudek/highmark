Health Plan Operations · AI portfolio decision discipline

AI opportunity is abundant.
Operating value is selective.

The work is not generating more use cases. It is deciding which workflow problems deserve investment, which evidence is authoritative, which judgments remain human, who owns adoption, and what proof advances—or stops—the work.

Prior authorization completeness

  1. 01

    Workflow consequence

    Incomplete or misrouted submissions create avoidable work before clinical review.

    Focus the intervention on completeness and routing—not coverage judgment.

  2. 02

    Evidence readiness

    Requirements, submitted documentation, and system context must remain traceable.

    Ground every assist in authoritative policy and submission evidence.

  3. 03

    Human authority

    Qualified people retain coverage authority.

    AI may prepare, explain, and route; it does not decide coverage.

    Authority signed
  4. 04

    Operating ownership

    Utilization Management owns workflow change, exceptions, training, and feedback.

    A proof has no value without an accountable operating owner.

  5. 05

    Proof standard

    Completeness, avoidable rework, turnaround, quality, and adoption must improve together.

    Advance only as a bounded proof with explicit scale and stop criteria.

03 · First 90 days

Earn the right to prioritize.

The first deliverable is not a grand roadmap. It is a trusted, evidence-backed way to make the next set of decisions better.

Days 1–30

Read the work

Map current initiatives, workflows, systems, data, policy constraints, human authority, ownership, frontline friction, and existing measures. Listen before prescribing.

  • Workflow and stakeholder maps
  • Current portfolio inventory
  • Evidence and authority gaps
Days 31–60

Charter the decisions

Establish a common opportunity record and review cadence so business, technical, governance, adoption, and value partners see the same decision.

  • Opportunity charters
  • Proof and stop criteria
  • Adoption and measurement plans
Days 61–90

Run controlled proof

Advance one or two bounded opportunities, instrument the workflow, capture user feedback, surface exceptions, and return learning to the portfolio.

  • Controlled proof briefs
  • Operating telemetry
  • Scale, hold, or stop review
Review the complete 90-day entry plan
